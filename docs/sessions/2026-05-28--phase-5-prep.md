## Decisions (Q1-Q8)

### Q1 + Q2 — Tier cluster naming

**Q1 (naming philosophy / full set)**: Brandon chose the
**discipline-centric set** (the Explore agent's recommendation) over
"rename tools only", "outcome-centric", and "keep current" — despite
the flagged triple-"research" collision. Signals he wants the names
to telegraph the auditable-research-artifact identity Phase 6a-6e is
building toward.

**Q2 (exact strings, within discipline-centric)**: chose the
**tightened** variant — drop the `research-` prefix from the tools
cluster. Final tier slugs:

| Slug | Was | Anchors |
|---|---|---|
| `publishing-infrastructure` | `tools` | book-scaffold-astro, deploy-workflows, runpod-deploy |
| `published-research` | `books-and-guides` | guides, guides-experimentation, book-template-astro, claude-books; mathematical-guides sub-entry |
| `pricing-decision-systems` | (keep) | annuity-pricing, Julia stack, insurance-ai-toolkit |
| `research-infrastructure` | (keep) | research-kb, research-agent, research_toolkit |

**Reframe that resolved the collision**: `publishing-infrastructure`
↔ `research-infrastructure` ↔ `published-research` reads as a
deliberate **infra/output triad** (infrastructure for publishing,
infrastructure for research, and the published research itself),
not monotonous repetition. Also shortened the 34-char
`research-publishing-infrastructure` to 25 chars. Reinforces the
build-to-learn identity locked in the prior ambiguity-resolution
session ("I build infra for two distinct workflows — shipping and
research").

### Q3 — Tier rendering shape + ordering

Reading the code surfaced a fork that "ordering" was hiding: the
homepage "Where this is going" section renders **projects as a flat
`<ul>`**, not cluster tiles; and `/work/[slug].astro` getStaticPaths
filters `section === 'now'`, so the 4 new clusters wouldn't get pages
without a change. Also a framing tension:
`publishing-infrastructure` + `research-infrastructure` are
**shipped, not aspirational**, so they must NOT fold under "Where
this is going."

**Decision**: **Compact tiles** — add a `compact` variant to
`ClusterCard` (hide artifact-list + screenshot placeholder), render
the 4 tiers as a lighter row under a NEW **"More work"** section
heading (distinct from the aspirational "Where this is going").
Delivers the genuine lead/tier hierarchy chosen in the prior session
(vs the 7-equal-cards explicitly rejected). All options required the
`/work/[slug]` getStaticPaths change.

**Ordering**: readiness-first (folded into Q3, adjustable via the
`order` field): `published-research` (guides live) →
`research-infrastructure` (live `/lab` demo) →
`publishing-infrastructure` (npm-published) →
`pricing-decision-systems` (dormant).

### Q4 — `research-infrastructure` cluster entries

**All three**, research-kb as anchor: `research-kb` (order 1) →
`research-agent` (order 2) → `research_toolkit` (order 3). Forms a
genuine stack story (data/search → agent QA → workflow skills). The
`tools`→`publishing-infrastructure` rename resolved the cross-cluster
ambiguity — `research_toolkit` clearly isn't a deploy/publish tool,
so it homes here. The 22 dogfooding orphans handled separately (Q6).

### Q5 — `research-infrastructure` copy framing

**"Enabling system" framing**: copy positions these as
infrastructure built to accelerate Brandon's OWN research/learning
— the spine under the build-to-learn work, powering the live
`/lab/research-graph` demo + study-notes pipeline. Resolves the
identity-pull warning (`portfolio-candidates.md:170-174`) actively
rather than dodging it; satisfies the `website-decision-map.md` §4
claim-safety note ("Avoid implying a production SaaS if the system
is personal/research infra"). Carries into the homepage tile
one-liner AND the `/work/research-infrastructure` page intro.

### Q6 — 22 dogfooding-orphan methodology page

**Defer to a focused Phase 6 follow-on**. The page's value depends
on narrative quality + a content-safety pass on what the local
dogfooding dirs expose (WIP, not all pushed) — neither fits this
session's scope, and it overlaps the existing `/how-this-was-made`
page + Phase 6a process-as-artifact work. Cluster ships with a
"what's next" hook pointing at it.
