## Gotchas discovered this session

Worth remembering because they bit us and the fixes weren't obvious:

1. **Cross-repo reusable workflow callers need an explicit `permissions:`
   block.** The default `GITHUB_TOKEN` permissions can be too narrow to
   satisfy the reusable workflow's declared minimums, producing
   `startup_failure` with the unhelpful message "This run likely failed
   because of a workflow file issue." Captured in
   `deploy-workflows/README.md`.

2. **`gh secret set NAME` syntax foot-gun.** The value is read from stdin
   at the interactive prompt; the argument after `gh secret set` is the
   *name*, not the value. Pasting the value where the name belongs creates
   a secret whose name leaks the credential.

3. **Workers Builds wizard ≠ Cloudflare Pages.** Cloudflare's dashboard
   funnels new users into the Workers Builds product, which uses
   `wrangler deploy` and `wrangler.jsonc` — *not* the legacy Pages product
   that the original deploy workflow targeted. Result: must explicitly back
   out of the Workers Builds wizard and avoid double-pipeline configs.

4. **Astro 6 strips HTML comments by default.** A `<!-- marker -->` won't
   survive the build. For a deploy round-trip marker, use a hidden element
   with a `data-*` attribute (`<div hidden data-deploy-rt="..."></div>`)
   instead.
