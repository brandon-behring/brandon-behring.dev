## Gotchas / notes (durable)

- **Worktree-off-`origin/main`** for every consumer (all had WIP or dirty main) — never touched a
  user checkout. All 5 worktrees removed at close; scaffold stays on `feat/v4.26-book-aware-nav`.
- **`.venv` pre-commit hook shim** (global `core.hooksPath=~/.git-hooks` → `~/.local/bin/pre-commit`
  resolves `$repo/.venv/bin/pre-commit` then PATH): a worktree has no `.venv`, so commit/push need a
  real `pre-commit` on PATH — `PATH="<a repo>/.venv/bin:$PATH"` ([[scaffold-consumer-fix-pattern]]).
  guides has no `.pre-commit-config` → no shim. dml's python gates (tier1/tier2 pytest, no file
  filter) are orthogonal to a `web/` bump and can't run in a venv-less worktree → `SKIP`-ped
  (`SKIP=gitleaks,unit-tests,integration-test-suite`; global gitleaks still scanned; python is
  byte-identical to the green base, so the gates were vacuous here).
- **dml** has **dual `origin` remotes** (brandon-behring + brandonmbehring-dev, SSH) — the branch
  pushed to both; the squash-**merge** landed only on brandon-behring (the dev-account `main` mirror
  is now one commit behind — its sync is the [[brandonmbehring-dev-second-account]] / #5 concern).
- **Cross-repo issue refs** in a consumer's commit/PR must be qualified (`book-scaffold-astro#162`),
  else GitHub mislinks to that consumer's own #162. (Amended the claude-books commit for this.)
- Benign: unused `ToolFilter`/`VersionSelector` island JS still bundles into the hub's `_astro/`
  (Astro emits imported island code) but is unreferenced/unloaded — scaffold-level, not the hub's.
